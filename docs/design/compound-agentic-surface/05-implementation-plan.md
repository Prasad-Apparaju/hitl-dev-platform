# Compound Agentic Surface (#10): Implementation Plan — 2.2.0

> Build sequencing for the **design-review-concluded** #10 core (HLD `01-design.md` v3.3, LLD `03-lld.md`,
> ADRs `02-adrs.md`). #10 ships **first** as 2.2.0; the Advisor (#35) follows. This is the *how-to-build*
> roadmap — it adds no design decisions; every artifact traces to an LLD §.

## 0. Principles (carried from the review)

- **Schema-first.** The manifest schema (#13) is the contract everything else validates; it lands before any
  validator.
- **TDD, per-rule.** Each validator is written against a **per-rule conformance fixture** (a manifest that
  should pass + one that should fail *on that exact rule*) before the check logic. The suite is the spec.
- **Verify by running the artifact, not by reading it** (the load-bearing lesson from 12 review rounds): every
  check is exercised on a real manifest; the **showcase manifest must actually fail its own checks** where the
  acceptance criteria say it should (HLD §13.3).
- **Source in `ai/claude/` / `ai/shared/` / `ci/` / `tools/`.** The plugin repo is generated by
  `tools/scripts/make-release.sh` — never hand-edit the built copy under `.claude/`. (Schema source of truth:
  `ai/claude/generate-docs/templates/system-manifest.schema.yaml`.)
- **Mirror the existing `ci/` pattern** (`ci/manifest-drift/`, `ci/preflight/`): one dir per validator family,
  `check_*.py` + `test_check_*.py`, plain-Python, fail-closed, self-contained.
- **Additive + per-check activation (ADR-13).** A legacy/deterministic manifest must validate with **no new
  registry**; each check activates only on its own data (LLD §6.0). This is acceptance criterion 1 and gets a
  test first.
- **Governs, not runtime.** No engine, broker, dashboard, or adapter *execution* ships (that's #42/#21).

## 3. Phase B — #16 Validators + gates (the core)

**File:** `ci/manifest-agentic/check_manifest_agentic.py` + `test_check_manifest_agentic.py`, plus
`tools/manifest-agentic/gen_baseline_evals.py`, `manifest-waivers.yaml`, and the eval `index.yaml`/`waivers.yaml`.

**The 17 checks (LLD §6), each with its activation predicate (§6.0) and a per-rule fixture:**

| Check | Activates on |
|---|---|
| `check_manifest_agentic` (orchestrator) | any new field present |
| `check_classification` | a domain with `kind` |
| `check_references` | typed `interactions` |
| `check_topology` | `interactions` + `orchestration` (cycle bound) |
| `check_boundary_legs` | an interaction touching an agent |
| `check_authorization` | `interactions.authorization` |
| `check_capabilities` | a domain with `uses`/`identity`/agent `kind` |
| `check_scope_grammar` | any capability scope string |
| `check_async` | `kind: event`/async/saga data |
| `check_memory` | a domain with memory `uses` |
| `check_lifecycle` | `long_running: true` |
| `check_deep_agent` | `kind: deep_agent` |
| `check_policy_refs` | any `PolicyRef` |
| `check_saga` | a declared `saga` |
| `check_compensation_gap` | ≥2 side-effecting interactions, no covering saga (**advisory**, never blocks) |
| `check_eval_coverage` | an agent domain / `e2e` segment |
| `check_observability` | any agentic system (**floor gate**, §6.17) |

**Order of implementation within B (each red→green with its fixture):**
1. Activation harness + `check_manifest_agentic` skeleton + acceptance-criterion-1 test (legacy/deterministic → no registry).
2. Graph-integrity family: `check_classification`, `check_references`, `check_topology`, `check_scope_grammar`.
3. Trust/privilege: `check_boundary_legs`, `check_authorization`, `check_capabilities` (over/under/ceiling), `check_policy_refs`.
4. Reliability/state: `check_async`, `check_memory`, `check_lifecycle`, `check_deep_agent`, `check_saga`, `check_compensation_gap` (advisory).
5. Coverage/floor: `check_eval_coverage` (per-agent + one e2e; `baseline_only` above Tier 1 fails), `check_observability` (floor gate + tier-scaled attrs, §6.17.1).
6. `manifest-waivers.yaml` fail-closed schema (every blocking check consults it) + `gen_baseline_evals.py`.

**Done:** the **showcase manifest** (HLD §13.3) blocks exactly the declared cases (untrusted agent leg,
unbounded agent endpoint, over/under/ceiling capability, unmatched memory write, unauthorized caller), the
two-side-effect flow emits the **advisory** (not a block), and an agentic manifest with no `observability`
block is blocked. `SEP-PAIR` (a call + an event between the same pair) validates.

## 7.5 Round-fable adversarial review — dispositions (2026-07-23)

A cold Fable subagent reviewed the implementation by execution and returned REVISIONS REQUIRED. Dispositions:

- **F1 (CRITICAL, fixed):** an unknown `kind`/enum silently fell through and deactivated governance checks
  (fail-*open*). Added **`check_schema`** — a fail-closed gate that runs first (on any compound content) and
  blocks `unknown_enum` for every load-bearing enum. A typo'd `kind: agent` now blocks, not passes.
- **F2 (fixed):** `unknown_field` was advertised but unimplemented. `check_schema` now rejects unknown keys in
  every known structure. Both `unknown_enum`/`unknown_field` are non-waivable.
- **F3 (fixed):** a matching-but-malformed waiver row crashed (exit 1). Now the whole waiver file is distrusted
  on any schema error (no row suppresses), `_waived`/`_eval_waived` are exception-guarded, and a crashing check
  is caught in `run()` as a fail-closed `schema_invalid` blocker.
- **F4 (fixed):** `gen_baseline_evals.py` (CR-20) and an observability posture view now ship; the
  `check-domain-boundary.sh` extension is **superseded** by the Python validators (§1, documented).
- **F5 (fixed):** the dead `assert True` waiver case is now a real tier/lapse assertion; per-rule fixtures added
  for the schema gate and the previously-thin rules (event ref, memory/lifecycle/deep-agent/saga sub-rules,
  observability convention/console). Suite: 43 → 52.
- **F6 (fixed):** `eval_console.ref: service:<x>` now must resolve to a declared `class: service` capability —
  no more evidence-theater bare strings.
- **F7 (accepted deviations, documented):** memory↔uses reconciliation is exact-scope (stricter than the LLD's
  `memory:<store>` capability route — no false negatives, but skips the memory ceiling); the §6.0 registry
  column lists advisory maxima (code loads only what it resolves); `facade_ref` is colon-form in code/showcase;
  `event_projection_mismatch` is covered by the generator's regenerate-and-diff rather than a standalone check;
  no CI workflow invokes the tools yet (repo convention — invocation lives in the workflow docs).
