# Agentic Design Advisor (#35 / FR-28): Implementation Plan

> Build sequencing for the **design-review-concluded** Advisor (v4.1: HLD `01-design.md`, ADRs `02-adrs.md`,
> LLD `04-lld.md`, test-plan `03-test-plan.md`). Ships **after** #10 (2.2.0 released); targets the 2.x line.
> This is the *how-to-build* roadmap — no new design decisions; every artifact traces to an LLD §.

## 0. Principles (carried from #10)

- **The Advisor recommends + records + hands off — it authors NO manifest field.** A human authors the
  manifest; #10 validates it. The hard boundary the feature exists to hold.
- **Deterministic core, TDD.** `compose.py`/`records.py` are pure Python over a scenario record; each rule
  gets a fixture (the test plan is the spec). Verify by running.
- **One intake command** (`hitl:agentic-intake`); the lenses are report **sections**, not 8 commands.
- **Source in `ai/…` / `tools/…` / `ci/…`;** the plugin is generated by the plugin repo's `build.sh` (add the
  advisor tooling to `shared/…` at release, mirroring `manifest-agentic`).
- **Neutral handoff:** `agentic-design-handoff.yaml` carries `role` + `proposed_kind` + neutral
  `connections`/`transport` + recommendation IDs + inline `target_path_hint`s — **no manifest field, not even
  `kind`** (round-9 B2).

## 3. Wave detail

- **A · compose.py (§4):** `LENSES`, `DEPENDS`, `any_agent`/`any_async`, `relevant`, `recommended_floor`
  (membership = safety factors incl. async; NO tier input, NO computed depth), `topo_order` (Kahn, first-ready
  in LENSES order — reused from #10), `compose`. Fixtures LOW/HIGH/ASYNC per test-plan §4; FLOOR-DET,
  PRUNE-DEPLOY, ASYNC-RELIABILITY.
- **B · catalog.yaml (§2):** one entry per question with `lens`/`role`/`ask_when`/`question`/`options`/
  `guidance`/`consequence` (tagged-union). Lint: every ADV-2 lens has ≥1 entry (CAT-COVERAGE); every option
  keyed or `*` (CAT-SCHEMA); `ask_when` parses (CAT-ASK-WHEN); every consequence target resolves; no
  `manifest_field` kind.
- **C · records.py (§7):** read/write `.hitl/agentic-state.yaml`; generate `agentic-decisions.md`
  (regenerate-and-diff, REC-GEN); generate the neutral `agentic-design-handoff.yaml` (HANDOFF-NEUTRAL,
  NO-AUTHOR — no manifest field/kind); rerun reconcile (RERUN-RECONCILE — recompute + confirm-before-write,
  skips not waivers).
- **D · render_map.py (§6):** `render(scenario, composed) -> {terminal, mermaid}`; node vocabulary by the
  elicited `role` enum (ROLE-TOTAL — required, single-valued, directly elicited); getting/available/not-needed
  breakdown = `LENSES − included` with reasons.
- **E · intake + integration (§3/§8):** the skill orchestrates the pieces; `pm-design-feature`/`AGENTS.md`
  gate→probe→route into `hitl:agentic-intake` on the compound branch (reconcile with #10's existing branch:
  the Advisor recommends first, then a human authors the manifest #10 validates).
- **F · docs + e2e + packaging (§9):** worked example, `docs/patterns`/playbook, E2E-INTAKE/ROUTE/MAP/DEPLOY;
  extend the plugin `build.sh` to ship `tools/agentic-advisor` + the catalog + the skill.

## 4. Definition of done

Every ADV-n met by the mechanism it names; each `ci/agentic-advisor` suite green; the worked example produces
a neutral handoff with **no** manifest field; `compose` is deterministic (a re-run reproduces the report);
routing is external + non-circular; the Advisor authors nothing #10 validates. Deferred (#43): rich HTML +
combined live map. Then a cold review before release on the 2.x line.
