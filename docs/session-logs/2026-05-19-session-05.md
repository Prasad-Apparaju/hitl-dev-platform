## What Was Done

### Finding #1 (High): Fix three SKILL.md files with YAML frontmatter parse failures

`claude plugin validate` (first real runtime run with Claude Code 2.1.142) failed on three skills due to unquoted `description:` values containing `: ` (colon-space), which YAML interprets as a nested mapping key-value separator.

Affected source files:
- `ai/claude/ops/verify-scripts/SKILL.md:2` — contained `workflow: after Step 6`
- `ai/claude/architect/review-code/SKILL.md:2` — contained `assess: business logic`
- `ai/claude/pm/review-scope-change/SKILL.md:2` — contained `scope change: impact`

Fix: wrapped each description value in double quotes in the source SKILL.md files.

### Finding #2 (Medium): Fix mangled relative+CLAUDE_PLUGIN_ROOT links in pm-design-feature

The v7 normalization step replaced `shared/challenge-stance.md` anywhere it appeared, including inside relative Markdown links like `[label](../../../shared/challenge-stance.md)`. This produced invalid paths: `[label](../../../${CLAUDE_PLUGIN_ROOT}/shared/challenge-stance.md)`.

Affected source file: `ai/claude/pm/design-feature/SKILL.md` lines 84 and 92.

Fix: replaced relative Markdown link syntax with prose references using bare `shared/challenge-stance.md` paths in the source. The normalization correctly expands these to `${CLAUDE_PLUGIN_ROOT}/shared/challenge-stance.md` in the built plugin.

**Two guards added to build.sh:**
1. **Mangled-path guard**: `grep -rl '\.\./.*\${CLAUDE_PLUGIN_ROOT}'` fails the build if any relative+CLAUDE_PLUGIN_ROOT combination is found.
2. **YAML frontmatter guard**: Python script parses every `SKILL.md` frontmatter block with `yaml.safe_load` and fails the build on any parse error.

These guards caught zero issues on the fixed build (40 SKILL.md files OK, no mangled paths).

### Finding #3 (Low): Add `version: "1.0.0"` to plugin manifests

Added `"version": "1.0.0"` to `ai/claude/plugin/plugin.json` (source). Updated the build.sh manifest sync step to sync both `description` and `version` (was description-only before), so the dist `.claude-plugin/plugin.json` stays in sync.
