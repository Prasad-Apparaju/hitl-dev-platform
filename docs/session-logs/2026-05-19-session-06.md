## What Was Done

### Finding (Medium): Remove PyYAML dependency from build.sh

The v8 YAML frontmatter guard used `import yaml` (PyYAML), which is not part of the Python standard library. On machines without PyYAML installed, `build.sh` exited non-zero at the validation step with `ModuleNotFoundError: No module named 'yaml'`.

Fix: replaced the PyYAML frontmatter block entirely with a call to `claude plugin validate "$PLUGIN_DIR"`. This:
- Has no external Python dependencies
- Is the canonical validator (catches frontmatter, manifest, and skill layout issues)
- Prints a clear skip message if `claude` is not on PATH, rather than failing

The mangled-path grep guard (pure bash) was left unchanged.

## Runtime Validation (from v9 reviewer)

## Key Decision

Used `claude plugin validate` rather than adding a `requirements.txt` — keeping the build dependency-free is more important than running the check offline. Contributors without the Claude binary get a clear skip message.
