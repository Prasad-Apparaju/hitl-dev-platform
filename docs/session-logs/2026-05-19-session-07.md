**Environment note from reviewer**: on the test machine, plain `claude` resolves to `/usr/local/bin/claude` (a broken wrapper), while the working binary is at `/Users/Prasad_1/.local/bin/claude`. Running `build.sh` without prefixing `PATH` fails at the validation step with "Error: claude native binary not installed."

Fix: added `CLAUDE_BIN` env var override to `build.sh` and printed the resolved binary path before running validation, so contributors can immediately see which `claude` is being used.

```bash
CLAUDE_BIN="${CLAUDE_BIN:-$(command -v claude 2>/dev/null || true)}"
echo "  using: $CLAUDE_BIN"
"$CLAUDE_BIN" plugin validate "$PLUGIN_DIR"
```

Also added a tip in the skip message: `set CLAUDE_BIN=/path/to/claude to specify the binary explicitly`.
