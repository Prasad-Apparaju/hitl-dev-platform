# Contributing to HITL AI-Driven Development Platform

Thank you for your interest in contributing. This is an early-stage project, and contributions are welcome.

## Reporting Issues

Open a GitHub issue with:

- A clear description of the problem or suggestion
- Steps to reproduce (for bugs)
- Expected vs. actual behavior
- Your environment (OS, tool versions) if relevant

## Proposing Changes

1. **Fork** the repository
2. **Create a branch** from `main` (e.g., `fix/broken-link` or `feat/new-template`)
3. **Make your changes** -- keep commits focused and explain *why* in commit messages
4. **Open a pull request** against `main` with a description of what changed and why

### What makes a good PR

- One concern per PR (don't mix unrelated fixes)
- Update documentation if your change affects user-facing behavior
- Follow existing code style and conventions
- If adding a new template or skill, include a brief usage example

## Code of Conduct

Be respectful and constructive. This project follows the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) code of conduct.

## Early-Stage Notice

This project is in active early development. APIs, file structures, and workflows may change. If you're planning a large contribution, open an issue first to discuss the approach before investing significant effort.

## AI-Assisted Contributions

This project uses AI coding assistants extensively. If your contribution was AI-assisted, include a `Co-Authored-By` trailer in your commit message. All AI-generated code should include inline comments explaining non-obvious logic.
